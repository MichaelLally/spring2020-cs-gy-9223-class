CAN Message Decoder

The CAN Decoder accepts a CAN message as input in base2 or base10, as a numeric type (or string if you insist), and returns a JSON response of the decoded message according to structure of the CAN protocol.

All returned values are in base10, except for data_bin and data_word_x fields. For demonstration purposes, this decoder does write to the database, but this feature will be disabled in a future release as the primary purpose is to decode CAN data for debugging.
